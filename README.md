# 压缩与缓存断点

一个 SillyTavern 扩展，帮你解决长对话的两大烦恼：

- **上下文太长，token 烧得肉疼** → 一键把前文压缩成一段"记忆摘要"
- **每次回复都全量重算，又慢又贵** → 自动打缓存断点，让 Claude 复用算过的前缀

配合 [gproxy](https://github.com/LeenHawk/gproxy) 使用效果最佳。

---

## 安装

1. 打开 SillyTavern 的 **扩展面板** → **Install extension**
2. 粘贴本仓库地址：`https://github.com/LeenHawk/st-compress-cache`
3. 装好后刷新页面，扩展面板里会出现「**压缩与缓存断点**」设置抽屉

---

## 功能一：一键压缩前文

聊到几百楼，前面的剧情又不舍得删？让模型帮你"记笔记"：

- 在设置面板点 **「立即压缩」**，或者在输入框里敲 `/compress`（想指定条数就 `/compress 20`）
- 扩展会把最近 N 条消息交给当前模型，按你写的提示词总结成一段摘要，作为新消息写回聊天
- 勾选「压缩后隐藏原始消息」的话，被压缩的原文会自动灰显、不再发给模型——记录还在，token 省了

**可以自己调的东西**：压缩提示词（决定摘要风格）、默认压缩条数、摘要以什么角色写回、摘要开头的标记文字。

---

## 功能二：缓存断点

Claude 支持"缓存"：把算过的前文存起来，下次直接复用，费用降到原价的一折左右，速度也快得多。难点在于告诉它"缓存到哪为止"——这就是断点。

本扩展会在每次发送前，自动在三个位置打断点：

| 断点位置 | 什么时候有用 | 默认 |
|---|---|---|
| 上次压缩的摘要 | 压缩过一次之后，摘要就是稳定前缀的结尾 | 关 |
| 最近一条 AI 回复 | 日常聊天，最常用 | 开（5 分钟） |
| 你刚发的消息 | 配合上一条，覆盖到最新内容 | 开（5 分钟） |

每个断点可以单独开关，缓存时长可选 5 分钟 / 1 小时 / 渠道默认。

断点只在发送的那一刻悄悄加上，**不会写进你的聊天记录**，也不会出现在 AI 的回复里。

### 需要准备什么

- 请求要走 **gproxy**，并在 gproxy 对应渠道里打开 **Magic-string cache** 开关（[gproxy 文档](https://gproxy.leenhawk.com/guides/claude-caching/)）
- 就这一件事，其他都不用动

### 效果参考

实测 claude-haiku-4.5：第一条消息写入 7500 token 的缓存，第二条消息这 7500 token 全部直接复用、零重算。

## 常见问题

**压缩会把原文删掉吗？**
不会。原文只是灰显、不再发给模型，聊天记录里随时能看。不想隐藏的话把勾去掉即可。

**断点会弄脏我的存档吗？**
不会。断点标记只存在于发送瞬间的请求里，存档和界面都看不到它。

**不用 gproxy 能用吗？**
压缩功能照常能用。断点功能需要 gproxy 识别标记，没有 gproxy 的话请把断点设为「关闭」（SillyTavern 自带的按深度缓存可以作为替代）。

**压缩用的是哪个模型？**
你当前连接的模型。压缩这一次请求本身不会被打断点。
