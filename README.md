# 压缩与缓存断点

一个 SillyTavern 扩展，帮你解决长对话的三大烦恼：

- **上下文太长，token 烧得肉疼** → 一键把前文压缩成一段"记忆摘要"
- **每次回复都全量重算，又慢又贵** → 自动打缓存断点，让 Claude 复用算过的前缀
- **最后一条回复差点意思，重 roll 又是开盲盒** → 用一句话指令局部改写，只改不满意的部分

配合 [gproxy](https://github.com/LeenHawk/gproxy) 使用效果最佳。

---

## 安装

1. 打开 SillyTavern 的 **扩展面板** → **Install extension**
2. 粘贴本仓库地址：`https://github.com/LeenHawk/st-compress-cache`
3. 装好后刷新页面，扩展面板里会出现「**压缩与缓存断点**」设置抽屉

---

## 功能一：压缩前文

聊到几百楼，前面的剧情又不舍得删？让模型帮你"记笔记"。

**两种模式：**

- **手动模式（默认）**：想压的时候再压。三个入口任选——
  - 输入框旁的 **"选项"菜单**（就是"重新生成 / AI帮答 / 续写"那个菜单）里的 **「压缩上下文」**
  - 设置面板里的 **「立即压缩」** 按钮
  - 命令 `/compress`
- **自动模式**：在设置里打开后，你每发满 N 条消息（默认 10 条，编辑重发不算数），AI 回复完就自动压缩一次。开着自动模式也随时可以手动压。

两种模式压的范围都是**上一次压缩点之后的全部消息**——不重复压、不漏压。想只压最近几条也行：`/compress 20`，或在按钮旁的小框里填条数。

- 压缩结果作为一条"记忆摘要"写回聊天
- 勾选「压缩后隐藏原始消息」的话，被压缩的原文会自动灰显、不再发给模型——记录还在，token 省了

**可以自己调的东西**：压缩提示词（默认为英文通用提示词，摘要会跟随对话语言）、自动压缩的触发条数、摘要以什么角色写回、摘要开头的标记文字。

---

## 功能二：缓存断点

Claude 支持"缓存"：把算过的前文存起来，下次直接复用，费用降到原价的一折左右，速度也快得多。难点在于告诉它"缓存到哪为止"——这就是断点。

本扩展会在每次发送前，自动在三个位置打断点：

| 断点位置 | 什么时候有用 | 默认 |
|---|---|---|
| 上次压缩的摘要 | 压缩过一次之后，摘要就是稳定前缀的结尾 | 关 |
| 最近一条 AI 回复 | 日常聊天，最常用 | 开（5 分钟） |
| 你刚发的消息 | 配合上一条，覆盖到最新内容 | 开（5 分钟） |

每个断点可以单独开关，缓存时长可选 5 分钟 / 1 小时 / 渠道默认。

断点只在发送的那一刻悄悄加上，**不会写进你的聊天记录**，也不会出现在 AI 的回复里。

### 需要准备什么

- 请求要走 **gproxy**，并在 gproxy 对应渠道里打开 **Magic-string cache** 开关（[gproxy 文档](https://gproxy.leenhawk.com/guides/claude-caching/)）
- 就这一件事，其他都不用动

### 效果参考

实测 claude-haiku-4.5：第一条消息写入 7500 token 的缓存，第二条消息这 7500 token 全部直接复用、零重算。

## 功能三：改写上一条

最后一条 AI 回复整体不错，就是某段不满意？不用重 roll 开盲盒，也不用自己动手编辑。

**用法：**

1. 在输入框写下改写要求，比如"把结尾那段改含蓄一点"
2. 点 **"选项"菜单** 里的 **「改写上一条」**（或用命令 `/rewrite 把结尾那段改含蓄一点`）
3. 模型带着完整上下文改写最后一条回复，改完直接替换

**几个贴心的地方：**

- **指令不进聊天记录**——它只存在于那一次请求里
- **原文自动存为 swipe**——改坏了左滑就能找回
- **默认局部替换模式**：模型只输出改动的片段（搜索替换块），扩展在本地应用——省输出 token，且不会顺手改掉你满意的部分。在设置里可以关掉，关掉后改为整条重写
- **缓存友好**：改写请求只在最后一条用户消息上打断点（和上一轮位置相同），前面的缓存前缀纯命中、零新写入；需要重算的只有被改写的回复和你的指令。改写后下一轮发送，也只有这条新回复需要重新写入缓存——这已经是内容变更下的理论最优，成本和 swipe 重 roll 相同，但改动可控得多

## 常见问题

**压缩会把原文删掉吗？**
不会。原文只是灰显、不再发给模型，聊天记录里随时能看。不想隐藏的话把勾去掉即可。

**断点会弄脏我的存档吗？**
不会。断点标记只存在于发送瞬间的请求里，存档和界面都看不到它。

**不用 gproxy 能用吗？**
压缩功能照常能用。断点功能需要 gproxy 识别标记，没有 gproxy 的话请把断点设为「关闭」（SillyTavern 自带的按深度缓存可以作为替代）。

**预设在对话记录后面有固定提示词（世界书@D、越狱、总结指令之类），断点会打到它们上面吗？**
不会。断点只会落在真实的聊天楼层上（最后一条 AI 回复 / 你的输入 / 压缩摘要），预设条目碰不到。这些尾部提示词位于断点之后，不会进入缓存前缀，也就不会导致缓存失效。唯一的例外：使用 OpenAI 兼容（custom）源、且"提示词后处理"选了**合并/严格类**模式时，尾部同角色提示词会被拼进你的输入消息里，「输入消息」断点会因此每轮失效——扩展检测到这种配置会弹警告。解决办法：后处理选"无"，或使用 Claude 源（其合并方式不受影响）。

**压缩用的是哪个模型？**
你当前连接的模型。压缩这一次请求本身不会被打断点。

**改写用的是哪个模型？会带完整上下文吗？**
你当前连接的模型，走完整的提示词构建管线（预设、世界书、全部历史都在），所以改写能贴合剧情。改写指令注入在上下文末尾，不会破坏缓存前缀。

**局部替换模式失败了怎么办？**
如果模型输出的替换块和原文对不上，扩展会尽量容错（忽略首尾空白、空白差异）；完全对不上的块会跳过并提示，一个都没对上则原文保持不动、指令保留在输入框方便重试。也可以在设置里关掉局部替换模式，改用整条重写。
